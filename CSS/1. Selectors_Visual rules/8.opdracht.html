Cascade

Betreft voorrang van stijlregels of CSS-eigenschappen (d.w.z. de volgorde waarin eigenschappen worden toegepast op een html-element.
Het lost conflictsituaties op.

Wat gebeurt er als we een set stijlregels definiëren voor een specifieke tag in externe, interne en inline stijlen?

Het antwoord is: "Regel get cascade"

Wanneer we een set regels definiëren voor een specifieke tag in externe, interne en inline-stijl-
maakt de browser een virtuele declaratielijst door alle stijlregels samen te voegen volgens de cascase-regels-
en past vervolgens de samengevoegde virtuele declaratielijst toe op de gespecificeerde tag.

De regels van cascade zijn onder meer:

- latere eigenschappen (dichtstbijzijnde) overschrijven eerdere (verstliggend) eigenschappen.
- meer specifieke selectors (minder algemeen) overschrijven minder specifieke (meer generieke) selectors)

Inheritance

Inheritance behandelt hoe stijlen van een ouderelement naar het onderliggende element werden gezuiverd.
Bepaalde eigenschappen, zoals font-family, worden geërfd. Als je een font-family instelt op het body-element,
dan wordt het overgenomen door alle elementen binnen de body class = "" Hetzelfde geldt voor kleur,
maar niet voor achtergrond of hoogte, die standaard altijd transparant en automatisch zijn.
